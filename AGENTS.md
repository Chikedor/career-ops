# Career-Ops for Codex

Read `CODEX.md` for the Codex routing rules and operating constraints in this repo.

Key points:
- Reuse the existing modes, scripts, templates, and tracker flow. Do not create a second automation layer.
- Store user-specific customization in `config/profile.yml`, `modes/_profile.md`, or `article-digest.md`, never in `modes/_shared.md`.
- Treat `templates/states.yml` as the source of truth for statuses.
- Never submit an application on the user's behalf.

Setup and usage details live in `docs/CODEX.md` and `docs/SETUP.md`.
